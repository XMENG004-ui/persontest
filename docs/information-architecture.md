# 信息架构 & 页面流程

## 1. 用户旅程（小红书闭环）

| 阶段 | 触点 | 用户动作 | 系统/运营动作 |
| --- | --- | --- | --- |
| 1. 购买前 | 小红书商品详情页 | 了解测试主题、下单 | 强调“2 元内·黑暗人格探索”卖点，展示结果页示例图、免责声明概述 |
| 2. 购后触达 | 店铺私信 / 订单自动回复 | 收到 H5 链接 + 兑换说明 | 发送唯一链接（或有效期 Token），附使用指南与客服提示 |
| 3. 进入测试 | H5 首屏（封面页） | 浏览主题介绍、点击 CTA | 加载版本选择结果，记录匿名 sessionId |
| 4. 版本选择 | 版本选择页 | 选择基础/深度版，阅读用时说明 | 根据选择拉取对应题库与免责声明，设置题目总数、进度条 |
| 5. 作答过程 | 问卷页面（多阶段） | 完成题目，随时可返回上一题 | 保存当前题号 + 答案到 localStorage，保持进度条、剩余题目提示 |
| 6. 提交计算 | 提交态 + Loading | 点击“提交获取报告”，等候计算 | 调用评分逻辑，生成维度分/原型/文案，写入结果快照 |
| 7. 查看报告 | 结果页 | 浏览总分、雷达、维度解读、原型、亮点与提醒 | 支持长按保存、点击“复制文案/分享链接”，展示免责声明 |
| 8. 售后与复购 | 结果页 CTA / 底部 Banner | 复测、购买其他主题、联系客服 | 推送其他测试入口，保留“重新作答”“回到店铺”链接 |


## 2. 页面节点与跳转关系

```text
Landing / Cover
    ↓ CTA（开始）
Version Picker (基础 / 深度)
    ↙                ↘
Start Checklist      Start Checklist
    ↓                  ↓
Question Flow (根据版本题目数)
    ↓ Submit → Loading
Result Report
    ↙            ↘
Share / Save   复测 / 其他测试
```

### 节点说明

1. **Landing / Cover**  
   - 内容：主题标题、引导文案、视觉主图、开始按钮、快速卖点（匿名/深度/耗时）。  
   - 出口：CTA → 版本选择；顶部菜单/底栏去除，只保留必要 Logo + 客服入口。

2. **Version Picker**  
   - 内容：基础版与深度版卡片，题数、耗时、适合人群、价格（如果拆成两个 SKU）。  
   - 功能：选项切换、免责声明预览、开始按钮。  
   - 状态：用户点击某版本后写入 `selectedVersion` 并进入作答准备。

3. **Start Checklist / Guide**  
   - 内容：温馨提示（环境、时间、隐私、作答方式）、免责声明确认勾选。  
   - 只有勾选后才能进入题目，防止纠纷。

4. **Question Flow**  
   - UI：单题全屏 + 进度条 + 5 级李克特选项。  
   - 交互：前进/后退、保存进度、异常退出提醒。  
   - 数据：根据版本加载 `base.json` 或 `deep.json`，题目顺序固定（后期可洗牌）。

5. **Submit + Loading**  
   - 提交后展示静态 Loading，提示“正在解析你的黑暗指数”。  
   - 后端或前端脚本计算得分 → 生成报告数据。

6. **Result Report**  
   - 模块：总指数卡片、雷达、维度卡片、人格原型、亮点/风险、金句/建议、免责声明。  
   - CTA：保存长图、复制报告文案、再测一次、查看更多测试、联系客服。

7. **Share / Upsell Layer**（结果页内层）  
   - 分享浮层：二维码/链接、快捷文案。  
   - 复购 Banner：引导其他测试、礼包包或返回店铺。


## 3. 关键状态 & 数据流

| 状态 | 持久化 | 说明 |
| --- | --- | --- |
| `sessionId` | localStorage | 进入 H5 时生成 UUID，用于匿名跟踪未完成作答。 |
| `selectedVersion` | localStorage | 基础 / 深度，影响题库、进度条、价格文案。 |
| `answers[]` | localStorage + 内存 | 记录题号、选项值、时间戳，支持中途刷新恢复。 |
| `progress` | 内存派生 | 当前题序 / 总题数，用于进度条、按钮文案。 |
| `resultSnapshot` | localStorage | 计算完成后缓存报告，便于回看/分享。 |
| `consentAccepted` | localStorage | 记录免责声明勾选状态，防重复提示。 |


## 4. 布局适配策略

- **移动优先**：单列布局、固定顶部状态提示 + 底部主要 CTA。  
- **PC 兼容**：中心定宽（max 480-600px），左右留空保持暗色背景。  
- **共享组件**：Top Banner、版本卡片、题目卡片、CTA 按钮、结果卡片等在 Figma/Tailwind 中统一 token，方便换主题。
